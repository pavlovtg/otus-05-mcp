# 0013. Формат логирования сервисов

- Статус: принят
- Дата: 2026-05-27

## Контекст

Проект содержит несколько сервисов (`mcp`, `wiki-api`), работающих в Docker Compose. Логи пишутся в stdout и собираются через `docker compose logs`. Для удобства анализа и фильтрации логов при формировании отчётов необходим единый формат записей с явным указанием источника (сервиса).

## Решение

Все сервисы проекта используют единый формат лог-записи:

```
{Timestamp:YYYY-MM-DD HH:mm:ss} {LEVEL} [{service}] Message {Properties}
```

- `{Timestamp}` — дата и время в формате `YYYY-MM-DD HH:mm:ss`
- `{LEVEL}` — уровень логирования (`INF`, `WRN`, `ERR`, `DBG`)
- `{service}` — фиксированный идентификатор сервиса (`MCP`, `WikiAPI`)
- `Message` — краткое описание события
- `{Properties}` — структурированные свойства в формате `Key=Value`

## Последствия

- Единый формат упрощает фильтрацию логов по сервису (`grep "[MCP]"`, `grep "[WikiAPI]"`)
- Структурированные свойства `Key=Value` удобны для парсинга при формировании отчётов
- Все новые сервисы обязаны использовать данный формат и задавать идентификатор `{service}`
