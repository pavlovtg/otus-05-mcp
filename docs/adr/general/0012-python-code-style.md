# 0012. Code style для Python

- Статус: принят
- Дата: 2026-05-25

## Контекст

- В репозитории появился Python-код (e2e-тесты в `tests/mcp/`)
- Без зафиксированных правил code style агент и разработчики могут использовать разные соглашения

## Решение

Python-код в репозитории следует правилам:

- **Версия Python:** 3.12+
- **Отступы:** 4 пробела (PEP 8)
- **Именование функций, переменных:** `snake_case`
- **Именование классов:** `PascalCase`
- **Именование констант:** `UPPER_SNAKE_CASE`
- **Аннотации типов:** обязательны для публичных функций и фикстур
- **Строки документации:** на русском языке, однострочные для простых функций
- **Импорты:** стандартная библиотека → сторонние пакеты → локальные модули (разделены пустой строкой)
- **Максимальная длина строки:** 120 символов

## Последствия

- Код единообразен независимо от автора
- Аннотации типов улучшают читаемость тестов и помогают IDE
- Отступление от стандарта Google (2 пробела) — осознанное решение в пользу PEP 8
