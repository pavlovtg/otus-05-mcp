# 0001. Структура хранения ADR

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий растёт, архитектурных решений будет много
- Нужна навигация по доменам без поиска по плоскому списку
- Агент читает индекс ADR в начале каждой задачи — индекс должен быть компактным

## Решение

ADR хранятся в `docs/adr/<domain>/`, кросс-доменные — в `docs/adr/general/`; индекс всех ADR — `docs/adr/README.md` (одна строка на ADR).

## Последствия

- Навигация по доменам упрощается при росте числа ADR
- Агент читает только индекс, детали — по запросу (экономия токенов)
- Новые ADR требуют выбора домена при создании
