# 0011. E2E-тесты через pytest для MCP-сервисов

- Статус: принят
- Дата: 2026-05-25

## Контекст

- MCP-сервер требует проверки на реально запущенном сервисе (unit/integration тесты не покрывают полный стек)
- .NET-тесты (ADR-0008) не могут тестировать взаимодействие MCP-сервера с wiki-api через сеть
- Нужен отдельный уровень тестирования для проверки end-to-end сценариев

## Решение

- E2E-тесты располагаются в `tests/mcp/` в корне репозитория
- Инструмент: pytest (Python 3.12+)
- HTTP-клиент: httpx
- Тесты запускаются против реально запущенного MCP-сервера
- URL конфигурируется через `MCP_URL` (default: `http://localhost:5200`)
- В GitHub Actions: отдельный workflow `.github/workflows/mcp-e2e.yml`
  - Поднимает `wiki-api` и `mcp` через `docker compose up`
  - Запускает pytest после проверки готовности сервисов
  - Запускается при `push` и `pull_request`

## Последствия

- E2E-тесты в отдельном workflow — не усложняют существующий `dotnet-ci.yml` (ADR-0009)
- Требуют запущенного docker-compose — нельзя запустить без инфраструктуры
- Покрывают: `list_apis`, `search_apis`, `resources/list`, `api://{name}`
