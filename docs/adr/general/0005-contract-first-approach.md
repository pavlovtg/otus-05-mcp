# 0005. Contract-first подход для сервисов репозитория

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий содержит несколько сервисов с REST API
- Без фиксированного подхода контракты могут создаваться после реализации (code-first)
- Программные потребители (MCP-сервер и др.) не могут работать параллельно с разработкой без готового контракта

## Решение

Все сервисы репозитория используют contract-first подход: swagger-контракт создаётся в `docs/contracts/<service>/` до начала реализации.

## Последствия

- Контракт доступен потребителям до завершения реализации
- Разработка API и потребителей может вестись параллельно
- Требует дисциплины: реализация не должна опережать контракт
