# 0002. Структура директорий репозитория

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий растёт: планируется MCP-сервер, инфраструктурный код, учебные задания
- Без зафиксированной структуры каждый участник (и агент) размещает файлы произвольно
- Нужна структура, масштабируемая при добавлении новых компонентов

## Решение

Репозиторий использует следующую структуру директорий верхнего уровня:

- `/apps` — исполняемые компоненты (MCP-сервер и другие приложения)
- `/infrastructure` — инфраструктурный код (Docker, Kubernetes, CI/CD)
- `/docs` — документация проекта
- `/docs/ai-memory-bank` — memory bank AI-агента
- `/docs/adr` — архитектурные решения
- `/homework` — учебные задания OTUS

## Последствия

- Навигация по репозиторию упрощается при росте числа компонентов
- Агент и разработчики имеют единый контракт на размещение файлов
- Новые компоненты требуют выбора между `/apps` и `/infrastructure`
