# 0009. GitHub Actions CI для .NET сервисов

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий содержит .NET сервисы с требованиями к качеству (тесты, coverage ≥ 80%)
- Без автоматической проверки требования могут нарушаться при слиянии PR
- Нужен CI, блокирующий слияние PR при несоответствии требованиям

## Решение

GitHub Actions workflow `.github/workflows/dotnet-ci.yml` запускается при каждом `push` и `pull_request`.

Шаги workflow:

1. `dotnet build` — сборка всех .NET сервисов
2. `dotnet test` — запуск тестов с измерением code coverage
3. Проверка порога coverage ≥ 80% — при несоответствии workflow завершается с ошибкой

Branch protection rules в GitHub настраиваются так, что слияние PR заблокировано при падении CI workflow.

## Последствия

- Сборка, тесты и coverage проверяются автоматически при каждом изменении
- PR не может быть слит при падении любого шага CI
- Единый workflow для всех .NET сервисов — единая точка конфигурации CI
