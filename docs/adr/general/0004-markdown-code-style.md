# 0004. Code style для Markdown

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий содержит большое количество Markdown-файлов (ADR, specs, memory bank, шаблоны)
- Без правил форматирования файлы несовместимы между парсерами и нечитаемы в raw-виде
- Выбраны правила из markdownlint v0.40.0 как минимально необходимый набор

## Решение

Все Markdown-файлы в репозитории соответствуют следующим правилам:

- **MD022** — заголовки окружены пустыми строками (сверху и снизу), кроме заголовка в начале файла
- **MD047** — файл заканчивается ровно одним символом перевода строки (`\n`)
- **MD012** — не более одной пустой строки подряд (вне блоков кода)
- **MD032** — списки окружены пустыми строками (сверху и снизу), кроме начала/конца документа
- **MD034** — URL и email обёрнуты в угловые скобки (`<url>`), если не используются в синтаксисе `[text](url)`
- **MD009** — строки не заканчиваются пробелами (кроме двух пробелов для явного `<br>`)

## Последствия

- Markdown-файлы совместимы с большинством парсеров (включая kramdown)
- Шаблоны OpenSpec обновляются под эти правила
- Существующие файлы исправляются по мере работы с ними
- Требуется `.markdownlint.json` для автоматической проверки (отдельный change)
