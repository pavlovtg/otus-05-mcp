# 0014. Serilog как библиотека логирования для .NET-сервисов

- Статус: принят
- Дата: 2026-05-27

## Контекст

.NET-сервисы проекта (`mcp`, `wiki-api`) используют ASP.NET Core. Встроенный `ILogger<T>` поддерживает структурированное логирование, однако встроенный `SimpleConsoleFormatter` не позволяет задать произвольный шаблон вывода с кастомными полями (например, `{service}`). Для соответствия формату, зафиксированному в ADR-0013, необходим инструмент с поддержкой настраиваемого `outputTemplate`.

## Решение

Все .NET-сервисы проекта используют **Serilog** (`Serilog.AspNetCore`) в качестве провайдера логирования. Serilog подключается через `UseSerilog()` в `Program.cs` и настраивается с `outputTemplate`, соответствующим ADR-0013. Идентификатор сервиса задаётся через `Enrich.WithProperty("service", "...")`.

## Последствия

- Пакет `Serilog.AspNetCore` добавляется как зависимость в каждый .NET-сервис проекта
- Встроенный Console-провайдер логирования отключается (заменяется Serilog)
- Все новые .NET-сервисы обязаны использовать Serilog с настройкой `outputTemplate` по ADR-0013
- Serilog широко используется в .NET-экосистеме и хорошо поддерживается
