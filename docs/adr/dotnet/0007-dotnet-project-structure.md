# 0007. Структура .NET проектов в репозитории

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий содержит несколько .NET приложений
- Без стандарта каждый разработчик организует структуру по-своему
- Нужно разделить исходный код и тесты для упрощения Docker-сборки

## Решение

Каждое .NET приложение в репозитории следует структуре:

```
apps/<app>/
├── src/          — исходный код проекта(ов)
├── tests/        — тестовые проекты
└── <app>.slnx    — файл solution
```

- Исходные проекты: `apps/<app>/src/<ProjectName>/`
- Тестовые проекты: `apps/<app>/tests/<ProjectName>.Tests/`
- Формат solution: `.slnx` (XML-based, без GUID, поддерживается с .NET 9+)

## Последствия

- Единообразная структура упрощает навигацию и onboarding
- Разделение `src/tests` позволяет копировать только `src/` в Docker-образ
- `.slnx` требует .NET 9+ (используем .NET 10 — ADR-0006)
