# 0003. Code style для C#

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Проект разрабатывается на C# (.NET)
- Без зафиксированных правил code style агент и разработчики могут использовать разные соглашения
- Нужен минимальный набор правил для единообразия кода

## Решение

C# код в репозитории следует правилам:

- **Отступы:** табы (не пробелы)
- **Именование классов, методов, свойств, публичных полей:** `PascalCase`
- **Именование локальных переменных и параметров:** `camelCase`
- **Именование приватных полей:** `_camelCase` (префикс `_`)
- **Именование констант:** `PascalCase`
- **Именование интерфейсов:** `IPascalCase` (префикс `I`)
- **Один публичный тип на файл;** имя файла совпадает с именем типа

## Последствия

- Код единообразен независимо от автора (человек или агент)
- Табы позволяют каждому разработчику настроить визуальную ширину отступа
- Требуется `.editorconfig` для автоматического применения правил (отдельный change)
- Отступление от стандарта Microsoft (4 пробела) — осознанное решение команды
