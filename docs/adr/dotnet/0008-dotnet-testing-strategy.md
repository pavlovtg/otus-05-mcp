# 0008. Стратегия тестирования .NET сервисов

- Статус: принят
- Дата: 2026-05-25

## Контекст

- Репозиторий содержит .NET сервисы, требующие тестового покрытия
- Без стандарта тестирование может быть неполным или несогласованным между сервисами
- Нужен минимальный порог качества, проверяемый в CI

## Решение

Каждый .NET сервис покрывается тестами трёх уровней в одном тестовом проекте `<ProjectName>.Tests`:

- **Unit-тесты** — бизнес-логика изолированно, зависимости заменяются моками
- **Integration-тесты** — взаимодействие компонентов с реальными зависимостями (файловая система, БД и др.)
- **API-тесты** — полный HTTP-вызов через `WebApplicationFactory<TProgram>` из `Microsoft.AspNetCore.Mvc.Testing`

Минимальный порог code coverage: **80% line coverage**.

Тестовый проект располагается в `apps/<app>/tests/<ProjectName>.Tests/`.

## Последствия

- Один тестовый проект упрощает конфигурацию и запуск
- `WebApplicationFactory` позволяет тестировать полный HTTP-стек без запуска реального сервера
- Порог 80% проверяется в CI (ADR-0009) и блокирует слияние PR при несоответствии
