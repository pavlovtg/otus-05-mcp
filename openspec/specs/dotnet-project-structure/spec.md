## ADDED Requirements

### Requirement: Стандартная структура .NET приложения

Каждое .NET приложение в репозитории SHALL следовать структуре:

```
apps/<app>/
├── src/          — исходный код проекта(ов)
├── tests/        — тесты
└── <app>.slnx    — файл solution
```

#### Scenario: Создание нового .NET приложения

- **WHEN** разработчик создаёт новое .NET приложение в репозитории
- **THEN** приложение размещается в `apps/<app>/` с подпапками `src/`, `tests/` и файлом `<app>.slnx`

#### Scenario: Расположение исходного кода

- **WHEN** разработчик добавляет C#-проект
- **THEN** проект размещается в `apps/<app>/src/<ProjectName>/`

#### Scenario: Расположение тестов

- **WHEN** разработчик добавляет тестовый проект
- **THEN** тестовый проект размещается в `apps/<app>/tests/<ProjectName>.Tests/`

### Requirement: Расположение контрактов сервисов

Swagger-контракты сервисов репозитория SHALL храниться в `docs/contracts/<service>/`.

Каждый сервис SHALL иметь отдельную подпапку. Имя подпапки SHALL совпадать с именем сервиса в kebab-case.

#### Scenario: Добавление контракта нового сервиса

- **WHEN** разработчик создаёт swagger-контракт для сервиса `wiki`
- **THEN** файл размещается в `docs/contracts/wiki/wiki-api.yaml`

#### Scenario: Несколько версий контракта

- **WHEN** сервис имеет несколько версий API
- **THEN** каждая версия хранится отдельным файлом в `docs/contracts/<service>/`
