## Requirements

### Requirement: Resource api://{name}

MCP-сервер SHALL предоставлять MCP-ресурс с URI-шаблоном `api://{name}`, возвращающий полное содержимое Swagger YAML-файла для указанного API.

#### Scenario: Получение контракта

- **WHEN** MCP-клиент запрашивает ресурс `api://{name}` с корректным именем
- **THEN** сервер возвращает содержимое соответствующего Swagger YAML-файла

#### Scenario: Контракт не найден

- **WHEN** MCP-клиент запрашивает ресурс `api://{name}` с несуществующим именем
- **THEN** сервер возвращает ошибку с понятным сообщением

### Requirement: Список ресурсов

MCP-сервер SHALL возвращать список доступных ресурсов при запросе `resources/templates/list`.

Каждый ресурс SHALL содержать URI, имя и описание.

#### Scenario: Перечисление ресурсов

- **WHEN** MCP-клиент запрашивает `resources/templates/list`
- **THEN** сервер возвращает список всех доступных API-контрактов как MCP-ресурсы
