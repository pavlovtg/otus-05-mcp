## Зачем

AI-агенты в IDE не имеют доступа к актуальным контрактам внутренних сервисов, что затрудняет генерацию корректного кода интеграций. MCP-сервер предоставит агентам структурированный доступ к каталогу API через стандартный протокол.

## Что меняется

- Добавляется новый сервис `apps/mcp` — MCP-сервер на .NET 10
- MCP-сервер использует wiki-api как источник данных о контрактах
- Существующий `docker-compose.yaml` дополняется сервисом `mcp`

## Возможности

### Новые возможности

- `mcp-server`: MCP-сервер, предоставляющий tools и resources для доступа к каталогу API
- `mcp-tools-api-discovery`: инструменты для получения списка API и поиска по ним
- `mcp-resources-api-contract`: ресурс для получения полного контракта конкретного API

### Изменённые возможности

- `wiki-docker-deployment`: добавляется сервис `mcp` в существующий docker-compose

## Влияние

- Новый проект `apps/mcp/` в репозитории
- Зависимость от wiki-api (должен быть запущен и доступен)
- `infrastructure/docker-compose/docker-compose.yaml` — добавляется новый сервис
