## ADDED Requirements

### Requirement: Пустые строки вокруг заголовков (MD022)

Каждый заголовок Markdown SHALL быть окружён пустыми строками сверху и снизу, за исключением заголовка в начале файла.

#### Scenario: Заголовок в середине документа

- **WHEN** заголовок расположен не в начале файла
- **THEN** перед заголовком и после него стоит ровно одна пустая строка

#### Scenario: Заголовок в начале файла

- **WHEN** заголовок является первой строкой файла
- **THEN** пустая строка перед ним не требуется

### Requirement: Файл заканчивается переводом строки (MD047)

Каждый Markdown файл SHALL заканчиваться ровно одним символом перевода строки (`\n`).

#### Scenario: Конец файла

- **WHEN** файл сохраняется
- **THEN** последний символ файла — перевод строки

### Requirement: Не более одной пустой строки подряд (MD012)

Markdown документ SHALL содержать не более одной пустой строки подряд (вне блоков кода).

#### Scenario: Несколько пустых строк

- **WHEN** между абзацами стоят две и более пустые строки
- **THEN** это является нарушением; оставить ровно одну пустую строку

### Requirement: Пустые строки вокруг списков (MD032)

Каждый список (маркированный или нумерованный) SHALL быть окружён пустыми строками сверху и снизу, за исключением начала/конца документа.

#### Scenario: Список после абзаца

- **WHEN** список следует за текстовым абзацем
- **THEN** между абзацем и списком стоит пустая строка

#### Scenario: Список перед заголовком

- **WHEN** список предшествует заголовку
- **THEN** между списком и заголовком стоит пустая строка

### Requirement: URL в угловых скобках (MD034)

URL и email-адреса в Markdown SHALL быть обёрнуты в угловые скобки (`<url>`), если они не являются частью синтаксиса ссылки `[text](url)`.

#### Scenario: Прямая ссылка в тексте

- **WHEN** URL упоминается в тексте напрямую
- **THEN** URL обёрнут в угловые скобки: `<https://example.com>`

#### Scenario: Ссылка в синтаксисе Markdown

- **WHEN** URL используется как `[текст](url)`
- **THEN** угловые скобки не требуются

### Requirement: Нет trailing spaces (MD009)

Строки Markdown файла SHALL NOT заканчиваться пробелами или табами (за исключением двух пробелов для явного переноса строки `<br>`).

#### Scenario: Конец строки

- **WHEN** строка заканчивается одним или более пробелами (не два пробела для `<br>`)
- **THEN** это является нарушением; trailing spaces удаляются
