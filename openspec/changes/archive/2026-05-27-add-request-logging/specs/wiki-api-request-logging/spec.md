## ADDED Requirements

### Requirement: Логирование входящего запроса

Wiki API SHALL логировать каждый входящий HTTP-запрос с указанием метода, пути, входных параметров и HTTP-статуса ответа.

#### Scenario: Успешный запрос списка контрактов

- **WHEN** клиент вызывает `GET /api/contracts`
- **THEN** в stdout записывается лог-запись с полями: метод, путь, `Status=200`

#### Scenario: Успешный поиск контрактов

- **WHEN** клиент вызывает `GET /api/contracts/search?q=<query>`
- **THEN** в stdout записывается лог-запись с полями: метод, путь, параметр `q`, `Status=200`

#### Scenario: Успешное получение контракта по имени

- **WHEN** клиент вызывает `GET /api/contracts/{name}`
- **THEN** в stdout записывается лог-запись с полями: метод, путь, параметр `name`, `Status=200`

#### Scenario: Запрос с ошибкой клиента

- **WHEN** запрос завершается с HTTP 400 или 404
- **THEN** в stdout записывается лог-запись с полями: метод, путь, входные параметры, соответствующий HTTP-статус

#### Scenario: Внутренняя ошибка сервера

- **WHEN** обработка запроса завершается необработанным исключением
- **THEN** в stdout записывается лог-запись уровня Error с полями: метод, путь, входные параметры, `Status=500`, тип исключения

### Requirement: Параметры без секретов

Логирование MUST NOT включать секретные данные (токены, пароли, ключи API).

#### Scenario: Логирование параметра поиска

- **WHEN** клиент вызывает `GET /api/contracts/search?q=<query>`
- **THEN** значение `q` логируется как есть (не является секретом)
