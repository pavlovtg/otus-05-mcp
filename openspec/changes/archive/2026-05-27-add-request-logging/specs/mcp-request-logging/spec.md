## ADDED Requirements

### Requirement: Логирование вызова инструмента

MCP-сервер SHALL логировать каждый вызов инструмента с указанием имени инструмента, входных параметров и статуса выполнения.

#### Scenario: Успешный вызов инструмента

- **WHEN** клиент вызывает инструмент `list_apis` или `search_apis`
- **THEN** в stdout записывается лог-запись с полями: имя инструмента, входные параметры, `Status=success`

#### Scenario: Ошибка при вызове инструмента

- **WHEN** вызов инструмента завершается исключением
- **THEN** в stdout записывается лог-запись с полями: имя инструмента, входные параметры, `Status=error`, тип исключения

### Requirement: Логирование вызова ресурса

MCP-сервер SHALL логировать каждый запрос к ресурсу с указанием URI-шаблона, параметров и статуса выполнения.

#### Scenario: Успешный запрос ресурса

- **WHEN** клиент запрашивает ресурс `api://{name}`
- **THEN** в stdout записывается лог-запись с полями: URI-шаблон ресурса, параметр `name`, `Status=success`

#### Scenario: Ошибка при запросе ресурса

- **WHEN** запрос ресурса завершается исключением
- **THEN** в stdout записывается лог-запись с полями: URI-шаблон ресурса, параметр `name`, `Status=error`, тип исключения

### Requirement: Параметры без секретов

Логирование MUST NOT включать секретные данные (токены, пароли, ключи API).

#### Scenario: Логирование строки поиска

- **WHEN** вызывается инструмент `search_apis` с параметром `query`
- **THEN** значение `query` логируется как есть (не является секретом)
