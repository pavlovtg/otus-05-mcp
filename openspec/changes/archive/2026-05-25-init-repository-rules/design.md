## Context

- Проект: MCP-сервер на .NET, разрабатывается совместно с AI-ассистентом Cline
- Текущее состояние: пустой репозиторий без правил взаимодействия с агентом
- Cline читает `.clinerules/` автоматически при каждой сессии
- `.clineignore` ограничивает файлы, доступные агенту без явного запроса
- OpenSpec использует схему `spec-driven-with-adr` с путём ADR `adr/` (требует изменения)

## Goals / Non-Goals

**Goals:**
- Настроить язык взаимодействия с Cline (русский)
- Задать формат промтов для экономии токенов
- Инициализировать memory bank для сохранения контекста между сессиями
- Настроить систему ADR в `docs/adr/`
- Ограничить доступ агента к нерелевантным файлам
- Создать `.gitignore` для .NET проекта

**Non-Goals:**
- Настройка CI/CD
- Настройка branch protection rules
- Создание документации по MCP-серверу

## Decisions

### Расположение правил Cline

**Решение:** `.clinerules/` — отдельные файлы по темам (`language.md`, `prompts.md`, `memory-bank.md`, `adr.md`)

**Альтернатива:** один файл `.clinerules` — отклонено, сложнее поддерживать

**Обоснование:** модульная структура позволяет независимо обновлять каждое правило

### Структура ADR

**Решение:** `docs/adr/<domain>/` для доменных ADR, `docs/adr/general/` для кросс-доменных; индекс в `docs/adr/README.md`

**Альтернатива:** плоская структура `docs/adr/` — отклонено, плохо масштабируется

**Обоснование:** доменная структура упрощает навигацию при росте числа ADR

### Путь ADR в OpenSpec

**Решение:** изменить `generates` в `schema.yaml` с `"../../../adr/*.md"` на `"../../../docs/adr/**/*.md"`; агент определяет домен из контекста change и создаёт ADR в `docs/adr/<domain>/`

**Обоснование:** агент знает домен из контекста change и может самостоятельно выбрать правильную подпапку; кросс-доменные ADR помещаются в `docs/adr/general/`

### Шаблоны OpenSpec

**Решение:** обновить все шаблоны (`adr.md`, `proposal.md`, `design.md`, `spec.md`, `tasks.md`) — русский язык, компактный стиль

**Обоснование:** единый стиль снижает когнитивную нагрузку и экономит токены

### Memory bank

**Решение:** `docs/ai-memory-bank/` — стандартные 6 файлов, русский язык, компактный формат

**Обоснование:** стандартная структура memory bank совместима с большинством Cline-конфигураций

## Risks / Trade-offs

- [Изменение пути ADR в OpenSpec] → существующие ADR (если появятся до применения) окажутся в `adr/`; митигация: применить change до создания первого ADR
- [Компактный стиль шаблонов] → потеря части контекста в подсказках; митигация: сохранить ключевые инструкции, убрать только вводные фразы

## Migration Plan

1. Применить change (создать все файлы)
2. Обновить `schema.yaml` и шаблоны OpenSpec
3. Обновить skills/workflows — заменить ссылки `<repo>/adr/` на `<repo>/docs/adr/`
4. Проверить: создать тестовый ADR → должен появиться в `docs/adr/<domain>/`
