## ADDED Requirements

### Requirement: Web-интерфейс для просмотра контрактов

Сервис SHALL предоставлять web-интерфейс на базе официального Swagger UI (`swaggerapi/swagger-ui`) для просмотра swagger-контрактов из каталога.

Интерфейс SHALL быть доступен по корневому URL сервиса Swagger UI (порт настраивается в docker-compose).

#### Scenario: Открытие интерфейса

- **WHEN** пользователь открывает URL Swagger UI в браузере
- **THEN** отображается список доступных контрактов из wiki REST API

#### Scenario: Просмотр контракта

- **WHEN** пользователь выбирает контракт из списка
- **THEN** Swagger UI отображает полное описание контракта: эндпоинты, параметры, схемы

### Requirement: Только просмотр (read-only)

Интерфейс SHALL отображать контракты только для чтения — без возможности выполнять HTTP-запросы к описываемым сервисам.

#### Scenario: Отсутствие кнопки "Try it out"

- **WHEN** пользователь просматривает описание эндпоинта
- **THEN** кнопка "Try it out" отсутствует или отключена

### Requirement: Источник данных через wiki REST API

Swagger UI SHALL получать список контрактов и содержимое файлов через wiki REST API, а не напрямую из файловой системы.

#### Scenario: Загрузка списка контрактов

- **WHEN** Swagger UI инициализируется
- **THEN** он обращается к `GET /api/contracts` wiki-сервиса для получения списка доступных контрактов
