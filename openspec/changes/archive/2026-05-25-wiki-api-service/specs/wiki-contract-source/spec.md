## ADDED Requirements

### Requirement: Папка как источник контрактов

Сервис SHALL читать swagger-контракты из папки, путь к которой задаётся через переменную окружения `CONTRACTS_PATH`.

#### Scenario: Чтение контрактов из папки

- **WHEN** сервис запускается с `CONTRACTS_PATH=/data/contracts`
- **THEN** сервис читает все `.yaml` и `.yml` файлы из указанной папки

#### Scenario: Отсутствие переменной окружения

- **WHEN** переменная `CONTRACTS_PATH` не задана
- **THEN** сервис использует значение по умолчанию `./content`

### Requirement: Поддержка динамического обновления

Сервис SHALL отражать актуальное содержимое папки при каждом запросе — без перезапуска сервиса.

#### Scenario: Добавление нового файла

- **WHEN** в папку источника добавляется новый `.yaml` файл
- **THEN** следующий запрос `GET /api/contracts` возвращает обновлённый список, включающий новый файл

#### Scenario: Удаление файла

- **WHEN** из папки источника удаляется файл
- **THEN** следующий запрос `GET /api/contracts` не содержит удалённый файл

### Requirement: Монтирование через Docker volume

В docker-compose конфигурации папка с контрактами SHALL монтироваться как volume в контейнер wiki-сервиса.

#### Scenario: Монтирование папки

- **WHEN** docker-compose запускает wiki-сервис
- **THEN** папка `apps/wiki/content/` с хостовой машины смонтирована в контейнер по пути `CONTRACTS_PATH`
